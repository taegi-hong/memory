### 1. 메서드 시그니처 및 파라미터
```java
public List<T> importExcelChunk(String sheetName, InputStream is, int titleNum) throws Exception
```
* **`sheetName`**: 읽어올 엑셀 시트(Sheet)의 이름입니다. 비어있을 경우 첫 번째 시트(index 0)를 기본값으로 사용합니다.
* **`is`**: 엑셀 파일의 데이터가 담긴 `InputStream`입니다.
* **`titleNum`**: 엑셀 헤더(컬럼 제목)가 위치한 행 번호(0부터 시작)입니다. 데이터는 이 행의 다음 행(`titleNum + 1`)부터 읽기 시작합니다.

---

### 2. 주요 동작 흐름 및 세부 분석

#### ① 초기화 및 워크북 생성 (409 ~ 415라인)
* 임포트 모드(`Type.IMPORT`)로 설정하고, Apache POI의 `WorkbookFactory.create(is)`를 호출하여 엑셀 워크북(`wb`) 객체를 메모리에 생성합니다.
* 지정된 `sheetName`이 존재하지 않으면 `"파일 sheet가 존재하지 않습니다"` 에러(다국어 메시지 처리)를 던집니다.

#### ② 이미지/사진 데이터 추출 (416 ~ 422라인)
* 엑셀 버전(`HSSFWorkbook`인 구버전 `.xls` 또는 `XSSFWorkbook`인 신버전 `.xlsx`)에 따라 알맞은 헬퍼 메서드(`getSheetPictures03` 또는 `getSheetPictures07`)를 호출하여 시트에 포함된 이미지 데이터를 추출합니다. 
* 결과는 `rowNum_colNum` 형태의 키 값을 갖는 `Map<String, PictureData>` 형태로 반환되어 나중에 이미지 셀 데이터를 파싱할 때 사용됩니다.

#### ③ 헤더(타이틀) 매핑 정보 생성 (423 ~ 444라인)
* 헤더 행(`titleNum`)을 읽어와서 각 컬럼의 타이틀 텍스트와 해당 열의 인덱스(`i`)를 매핑한 `cellMap`을 생성합니다.
* 대상 클래스(`clazz`)의 필드 중 `@Excel` 어노테이션이 붙은 필드들을 가져와서(`getFields()`), 엑셀 컬럼명(`attr.name()`)과 헤더 컬럼명이 일치하는 열 인덱스를 찾아 `fieldsMap`에 담아둡니다.
  * 이를 통해 엑셀의 열 순서가 바뀌더라도 헤더 이름 기준으로 매핑이 올바르게 매칭되도록 보장합니다.

#### ④ 청크(Chunk) 단위 행 반복 처리 (446 ~ 447라인)
* 데이터 행(`titleNum + 1`)부터 마지막 행(`rows`)까지 **`chunkSize` 단위**로 끊어서 루프를 돕니다.
```java
for (int startRow = titleNum + 1; startRow <= rows; startRow += chunkSize) {
    int endRow = Math.min(startRow + chunkSize - 1, rows);
    // ...
}
```
* *참고*: 현재 구현 상으로는 이 루프 안에서 즉시 전체 행을 객체로 변환하여 하나의 `List<T>`에 모두 모아서 리턴하고 있으므로, 메모리 분할 보관 기능보다는 청크 단위의 반복 흐름을 정의한 형태입니다.

#### ⑤ 행별 데이터 변환 및 객체 바인딩 (448 ~ 516라인)
각 행을 순회하면서 객체를 채우는 핵심 로직이 진행됩니다.
* **빈 행 검사**: `isRowEmpty(row)`를 통해 내용이 없는 행은 건너뜁니다.
* **객체 생성**: 최초 1회 반사(Reflection)를 통해 지정한 제네릭 타입의 인스턴스를 동적으로 생성합니다 (`clazz.newInstance()`).
* **데이터 타입별 형변환(Type Conversion)**:
  * 셀 타입과 매핑 필드의 타입(`fieldType`)에 맞춰 적절한 변환 작업이 진행됩니다.
  * **String**: 소수점 뒤의 `.0` 제거 처리 및 `@Excel(dateFormat=...)` 어노테이션이 설정된 경우 날짜 형식 포맷팅 적용.
  * **Integer / Long / Double / Float / BigDecimal**: 문자열 숫자를 해당 자바 숫자 타입으로 형변환.
  * **Date**: 셀 값이 문자열일 경우 날짜 포맷 파싱, 숫자(Double)일 경우 POI의 `DateUtil.getJavaDate`를 활용해 날짜 변환.
  * **Boolean**: 문자열을 `boolean` 타입으로 변환.
* **어노테이션 기반 특수 변환 적용**:
  * **`targetAttr`**: 중첩 객체 구조(예: `user.roleId` 등)에 값을 매핑할 때 사용됩니다.
  * **`readConverterExp`**: 표현식 변환(예: `0=남자,1=여자`와 같이 엑셀의 텍스트 값을 실제 DB 코드값으로 역매핑).
  * **`dictType`**: 공통 코드(Dictionary)에 매핑된 값을 역조회하여 코드로 변환.
  * **`handler`**: 사용자 정의 포맷 핸들러 어댑터가 설정되어 있으면 이를 통한 후처리 수행.
  * **`ColumnType.IMAGE`**: 이미지 셀 데이터인 경우, ②단계에서 추출해둔 사진 바이너리 데이터를 `FileUtils.writeImportBytes(data)`로 서버 저장소에 쓰고, 저장된 파일의 주소/구분값을 필드 값으로 설정합니다.
* **Setter 호출**: 변환된 데이터를 `ReflectUtils.invokeSetter(entity, propertyName, val)`를 통해 최종적으로 객체에 주입합니다.
* 완성된 객체를 `list`에 추가합니다.

전체적인 엑셀 업로드(Import) 프로세스는 다음과 같이 크게 6단계 흐름으로 동작합니다.

### 📋 엑셀 업로드 프로세스 Flow

1. **[1단계] 클라이언트 파일 전송**:
   * 사용자가 프론트엔드(Vue 3) 화면에서 엑셀 파일을 선택하여 업로드하면, API 요청(MultipartFile)을 통해 서버(Spring Boot)로 파일이 전송됩니다.

2. **[2단계] Controller 수신 및 Service 위임**:
   * Controller가 요청을 받아 유효성을 1차 체크하고, 비즈니스 로직을 처리하는 Service 구현체(`ServiceImpl`)로 `MultipartFile` 혹은 `InputStream`을 넘겨줍니다.

3. **[3단계] ExcelUtil 초기화 및 워크북 생성**:
   * 서비스 단에서 `ExcelUtil`을 사용해 `importExcelChunk` 메서드를 호출합니다.
   * `WorkbookFactory.create(is)`를 통해 엑셀 데이터를 POI `Workbook` 객체로 로드하고, 시트 유효성 검사 및 임베디드 이미지 정보를 맵 형태로 사전 추출합니다.

4. **[4단계] 헤더 매핑 규칙 수립 (Metadata Mapping)**:
   * 엑셀 헤더 행의 제목들을 읽어 각 열 인덱스를 구하고, 변환 대상 객체(Entity)의 `@Excel` 어노테이션 메타데이터와 대조하여 **"엑셀 열 인덱스 ↔ 자바 객체 필드"** 간 매핑 맵(`fieldsMap`)을 생성합니다.

5. **[5단계] 행 순회 및 자바 객체 변환 (Data Parsing & Binding)**:
   * 실제 데이터가 시작되는 행부터 끝까지 청크/루프로 돌며 빈 행을 걸러낸 뒤 각 행마다 객체를 동적으로 생성(`clazz.newInstance()`)합니다.
   * 각 셀 데이터를 읽어 필드 타입에 맞게 변환하고, `@Excel` 설정에 따른 세부 옵션(날짜 포맷팅, 공통 코드/딕셔너리 명칭 역매핑, 서버 스토리지 이미지 업로드 등) 처리 후 Reflection으로 필드 값을 세팅합니다.

6. **[6단계] DB 영속화 및 결과 리턴**:
   * 파싱 완료된 자바 객체 리스트(`List<T>`)를 받아 서비스 레이어에서 DB에 일괄 저장(Insert/Update 등)하고, 처리 결과(성공 건수, 실패 건수 등)를 가공하여 최종 응답(`AjaxResult`)을 사용자에게 반환합니다.
